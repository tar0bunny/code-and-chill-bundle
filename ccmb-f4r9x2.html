Your cozy little coding home in the browser

pythononline.net is where you'll write and run your code — and the best part is it lives entirely in your browser. No downloads, no installations, no "why isn't this working on my laptop."

Just open it, write some code, and hit run.

Think of it like Google Docs, but for coding.

1

Go to pythononline.net

Recommended if you plan to use it more than once — your code won't disappear between sessions.

You're in!

The editor is on the left, the output appears on the right. You only need to know two things to get started.

Python cheat sheet

Just enough to get started

You don't need to read this before you start. Think of it as a reference card — flip back here whenever something in the project confuses you. It's not a test, it's a cheat sheet. That's literally what it's for. 🌸

Comments

Lines Python ignores — leave yourself notes

# this is a comment
# python won't run this line
output → (nothing — comments are invisible to Python)

Variables

Named containers that store information

name = "Ada"
age = 25
print(name)
print(age)
output → Ada
25
The = sign means "store this value," not "these are equal."

Data types

The kinds of information Python understands

"hello"string — text, always in quotes
42integer — a whole number
3.14float — a number with a decimal
True / Falseboolean — yes or no

Print statements

How you make Python show you something

print("hello, cozy world!")
print("I am", 25)
output → hello, cozy world!
I am 25

Math operators

The basics of doing calculations in Python

5 + 2addition→ 7
5 - 2subtraction→ 3
5 * 2multiplication→ 10
5 / 2division→ 2.5
5 ** 2exponent→ 25
5 % 2remainder→ 1

If / else

Making decisions in your code

mood = "tired"

if mood == "tired":
    print("go take a break")
elif mood == "happy":
    print("let's build something!")
else:
    print("grab a cozy drink first")
output → go take a break
Note the colon at the end of each condition, and the indent on the next line — Python is picky about both.

Loops

Repeating code without rewriting it

for i in range(3):
    print("cozy")
output → cozy
cozy
cozy

Functions

Write once, use whenever you need it

def greet(name):
    print("hello,", name)

greet("Ada")
greet("Taylor")
output → hello, Ada
hello, Taylor

Importing a library

Borrowing pre-written code

import random

print(random.choice([
  "tea",
  "coffee",
  "hot chocolate"
]))
output → coffee (random each time!)

Your first project

A quote generator — what do you need to hear right now?

You're going to build a quote generator. Pick your vibe from a menu, and get a message that feels like it was made just for you — right in the output panel.

It runs, it's interactive, and the quotes are completely yours to edit.

What you'll learn

Dictionaries

Your quotes are stored in a dictionary — like a labeled filing cabinet, one drawer per category.

Loops

A while True: loop keeps the app running until you decide to quit.

Randomness

random.choice() picks a surprise quote every time.

User input

The app waits for you to type a number, then responds — that's what makes it interactive.

✏️

Look for the ✏️ icon in the code — that's your signal that something is customizable! The quotes and category names are the main things to make your own. Find the quotes = { ... } section and swap in whatever you want each category to say. You can rename categories, add new ones, or delete ones you don't need. Don't forget to change "friend" in the greeting to your actual name too.

quotes.py

Copy & paste the below code into pythononline.net 
import random

# ── YOUR QUOTES — CUSTOMIZE THIS! ✏️ ──────────────
# Replace the text between the quotation marks
# with whatever you want. Each line needs a comma!

quotes = {

    # ✏️ rename this category by changing the text in quotes
    "✨ delulu": [
        "Of course it works out for you",      # ← your quote here
        "You're literally the main character",  # ← your quote here
        "Things always fall in your favor",     # ← your quote here
    ],

    # ✏️ rename this category by changing the text in quotes
    "☁️ comfort": [
        "It's okay to rest",                   # ← your quote here
        "You're doing the best you can",        # ← your quote here
        "You're going to be okay",              # ← your quote here
    ],
}

# ✏️ change "friend" to your name!
print("\nHi friend, what do you need to hear right now?")
print("Pick a category and get your quote ✨\n")

# shows the numbered list of categories
category_list = list(quotes.keys())
for i, category in enumerate(category_list, 1):
    print(f"  {i}. {category}")

# asks you to pick a number
choice = input("\nType a number and press Enter: ")

# picks a random quote from the category you chose
category = category_list[int(choice) - 1]
quote = random.choice(quotes[category])

print(f"\n{category}")
print(f"\n  {quote}\n")
print("✨ You needed this today\n")

Understanding your code

What the important lines are actually doing

You built something real — now let's break down what the key parts are doing. You don't need to memorize any of this. Just read it once and let it click.

import random

Importing a library

Line 1

This line borrows pre-written code from Python's random library. Libraries are collections of useful functions that other people already wrote and tested. The random library gives you tools for picking random items, which is exactly what you need to pick a surprise quote each time.

Think of it like borrowing a tool from a friend instead of building it yourself from scratch.
quotes = { "☁️ comfort": [...], ... }

The dictionary

Line 6

This is where all your quotes live. A dictionary stores information in key/value pairs — the category name is the key, and the list of quotes is the value. You can look up any category by name, which is exactly what the program does when you pick a number.

Think of it like a real dictionary: you look up a word (the key) and get a definition (the value).
category_list = list(quotes.keys())

Pulling out the category names

Line 27

.keys() grabs all the category names from your dictionary. Wrapping it in list() turns them into an ordered list so you can number them. This is how the app knows to show "1. ✨ delulu, 2. ☁️ comfort" and so on.

for i, category in enumerate(category_list, 1):

The numbered list loop

Line 28

This loops through each category and gives it a number at the same time. enumerate() is a handy built-in that counts as it loops — the 1 tells it to start counting from 1 instead of 0. Without this, you'd have to manually number everything.

choice = input("Type a number and press Enter: ")

Getting user input

Line 32

This pauses the program and waits for you to type something. Whatever you type gets stored in the variable choice so the program can use it to pick the right category.

quote = random.choice(quotes[category])

Picking a random quote

Line 36

quotes[category] looks up the list of quotes for whichever category you chose. random.choice() then picks one at random from that list. This is the line that makes the app feel alive — every run might give you something different.

It's the digital equivalent of pulling a fortune cookie slip from a bowl.

These patterns — dictionaries, loops, input, randomness — show up in almost every Python project you'll ever build. You just learned them by doing something real. 🌸

Code & Chill Bundle · Section 06

What to do when you're stuck

Getting stuck is part of coding, not a sign you're bad at it

Something's not working
First, ask

Is there a red error message?

Yes

Read the error

  • Read it top to bottom — focus on the last line
  • Copy the exact message and Google it
  • Add "Python" to your search
  • Found a fix? Try it and go back to start
No error

Check your code

  • Add a print() to see what Python thinks is happening
  • Check your indentation — Python is picky
  • Read your code out loud, line by line
Still stuck?

Take a breath and try these

  • Walk away for 10 minutes — seriously, it works
  • Ask an AI assistant to explain the error, not just fix it
  • Post in a beginner community (Reddit r/learnpython, Discord)
  • Search for a YouTube tutorial on the specific thing that's breaking
When it works

It works! You figured it out. 🎉

🌸

One last thing: when you fix a tricky bug, leave yourself a comment in the code explaining what the problem was and how you solved it. Future you will be very grateful.
